# Claude, ChatGPT and Google Antigravity: primary-source research for the lecture website

**Evidence checked:** 22 July 2026  
**Audience:** Australian translators and interpreters who may handle confidential client material  
**Publication rule:** Every product claim below links directly to a first-party vendor page. Anything that could not be established from a current first-party page is labelled **Unverified — do not publish as fact**.

## What this report is comparing

The three useful presentation-level comparisons are **Claude Cowork**, **ChatGPT Work**, and **Google Antigravity 2.0**. They are not exact equivalents. Claude Cowork and ChatGPT Work are designed for broad knowledge work, while Google describes Antigravity as an agentic development platform for building software and tools. ([Claude Cowork](https://support.claude.com/en/articles/13345190-get-started-with-claude-cowork), [ChatGPT Work](https://learn.chatgpt.com/docs/get-started-with-work), [Google Antigravity](https://antigravity.google/))

The coding-oriented modes also need to be named clearly. Claude Desktop has a **Code** area for Claude Code; the new ChatGPT desktop app has a separate **Codex** area; and Google offers Antigravity 2.0, Antigravity IDE, Antigravity CLI, and an Antigravity SDK. ([Claude Code desktop](https://code.claude.com/docs/en/desktop), [ChatGPT Work and Codex](https://help.openai.com/en/articles/20001275/), [Antigravity product family](https://antigravity.google/))

### Availability cautions worth keeping beside the links

- Claude Cowork is available on desktop for Pro, Max, Team and Enterprise. Anthropic also documents a gradual web/mobile beta rollout beginning with Max, so not every otherwise eligible account is guaranteed to see it immediately. ([Cowork surfaces](https://support.claude.com/en/articles/15520349-use-claude-cowork-on-web-desktop-and-mobile))
- ChatGPT Work is available in the desktop app across plans with plan-dependent limits; OpenAI’s current help page describes cloud Work on web/mobile for eligible paid plans and local-capability Work on desktop. The exact combination is still being rolled out, so the website should say “availability and limits depend on plan and surface,” not “identical everywhere.” ([Work and Codex](https://help.openai.com/en/articles/20001275/), [Work pricing](https://learn.chatgpt.com/docs/pricing))
- Antigravity for individuals uses personal Google accounts. Google points organisations to a separately governed Google Cloud enterprise offering rather than treating a personal Google AI subscription as an organisational account. ([Antigravity FAQ](https://antigravity.google/docs/faq), [Antigravity enterprise](https://antigravity.google/docs/enterprise))

## What each broad work mode can do

### Claude Cowork

Anthropic describes Cowork as an agentic mode that accepts an outcome, plans multi-step work, divides complex work into subtasks and can coordinate parallel sub-agents. It can run code and shell commands, work with authorised folders, use connectors and plugins, browse and act in Chrome, and create documents, PDFs, spreadsheets, presentations, charts and transformed datasets. ([Cowork guide](https://support.claude.com/en/articles/13345190-get-started-with-claude-cowork))

Cowork can also run scheduled tasks remotely and maintain projects with their own files, instructions and context. “Live artifacts” are persistent interactive HTML dashboards and are currently desktop-only. ([Cowork surfaces](https://support.claude.com/en/articles/15520349-use-claude-cowork-on-web-desktop-and-mobile), [live artifacts](https://support.claude.com/en/articles/14729249-use-live-artifacts-in-claude-cowork))

Cowork **computer use** is a separate research-preview capability that can click, type and navigate approved desktop applications. As checked, it is offered to Pro and Max users on macOS and Windows, not Team or Enterprise users. ([computer use](https://support.claude.com/en/articles/14128542-let-claude-use-your-computer-in-cowork))

### ChatGPT Work

OpenAI describes Work as the mode for substantial, multi-step everyday tasks that should end in a finished deliverable. Its official examples include presentations, comparison spreadsheets, analysis, recurring updates, workflows and transforming a brief into an output; OpenAI contrasts this with Chat for answers, explanations, brainstorming and short drafts. ([Work guide](https://learn.chatgpt.com/docs/get-started-with-work))

Work can use uploaded files, plugins and approved tools. On desktop, with permission, it can work with local files, local applications and the browser; cloud Work can continue remotely and sync across devices. ([Work guide](https://learn.chatgpt.com/docs/get-started-with-work), [Work and Codex](https://help.openai.com/en/articles/20001275/))

OpenAI says Work and Codex share core agentic capabilities but present different experiences: Work is arranged around everyday deliverables, while Codex is arranged around software repositories, terminals and development tools. ([Work guide](https://learn.chatgpt.com/docs/get-started-with-work), [Work and Codex](https://help.openai.com/en/articles/20001275/))

### Google Antigravity 2.0

Google describes Antigravity 2.0 as a command centre for directing multiple local agents across projects and workspaces. It can run parallel agents, create scheduled messages and work against local project folders; Google’s official codelab describes it as a standalone app distinct from the older full Antigravity IDE. ([Antigravity product page](https://antigravity.google/), [official codelab](https://codelabs.developers.google.com/getting-started-google-antigravity))

Antigravity agents can edit files, run terminal commands, use browser capabilities and connect to external tools through Model Context Protocol servers. These capabilities are governed by workspace, terminal, URL, browser and external-tool permissions. ([settings](https://antigravity.google/docs/settings), [permissions](https://antigravity.google/docs/permissions), [MCP integrations](https://antigravity.google/docs/mcp))

Antigravity is therefore the clearest of the three products to present as a **building environment**, rather than as a general chatbot. Google markets Gemini itself for chat, research, file upload and work inside Gmail, Docs and Sheets, while Antigravity is marketed for orchestrating coding agents. ([Google AI plans](https://one.google.com/about/google-ai-plans/), [Antigravity product page](https://antigravity.google/))

## Execution and isolation: what is actually happening

### Claude Cowork

Current Anthropic architecture documentation says Cowork runs remotely by default. Its agent loop and code run in a temporary isolated sandbox on Anthropic-managed infrastructure; each session has its own sandbox, which is destroyed at session end; outbound traffic passes through an enforced proxy; private/internal network addresses are inaccessible by default; and connector credentials remain outside the sandbox. ([Cowork architecture](https://support.claude.com/en/articles/14479288-claude-cowork-architecture-overview))

Anthropic also documents an older/existing local deployment. There, the agent loop and connected-folder operations run on the user’s device while code and shell commands run inside a dedicated Linux virtual machine built with Apple Virtualization.framework on macOS or Hyper-V on Windows. ([Cowork architecture](https://support.claude.com/en/articles/14479288-claude-cowork-architecture-overview))

Remote Cowork can access local folders only through a connected and open Claude Desktop app, but files opened in a remote session are processed on Anthropic servers. The sandbox protects the execution environment; it does not prevent the agent from using folders, services and actions that the user has authorised. ([Cowork surfaces](https://support.claude.com/en/articles/15520349-use-claude-cowork-on-web-desktop-and-mobile), [Cowork safety](https://support.claude.com/en/articles/13364135-use-claude-cowork-safely))

Cowork computer use has no sandbox separating Claude from an approved desktop application. Claude takes screenshots and can see information displayed in those apps; Anthropic specifically advises against using this preview with legal documents, medical information, financial accounts or apps containing other people’s personal information. ([computer use](https://support.claude.com/en/articles/14128542-let-claude-use-your-computer-in-cowork))

### ChatGPT Work and Codex

OpenAI’s local agent sandbox constrains file and network access for spawned commands. It uses native operating-system mechanisms: Seatbelt on macOS, bubblewrap on Linux/WSL and a native Windows sandbox. The recommended “Ask for approval” mode allows normal work within the chosen workspace and asks before internet access or access beyond it; “Full access” can edit any file and use the network without approval. ([sandboxing](https://learn.chatgpt.com/docs/sandboxing), [permission modes](https://learn.chatgpt.com/docs/permission-modes))

This sandbox is not a promise that every connected application or external tool is isolated. OpenAI separates command sandboxing from approval policy and warns that connected tools and network access expand what an agent can see and do. ([sandboxing](https://learn.chatgpt.com/docs/sandboxing), [agent security](https://learn.chatgpt.com/docs/agent-approvals-security))

Codex cloud creates an isolated OpenAI-managed container, checks out the repository, runs a setup step and then lets the agent work. Network access is enabled for setup and disabled during the agent phase by default, unless the user configures an allowlist or broader access; environment secrets are available during setup and removed before the agent phase. ([cloud environment](https://learn.chatgpt.com/docs/environments/cloud-environment), [internet access](https://learn.chatgpt.com/docs/cloud/internet-access))

OpenAI warns that enabling cloud internet access adds prompt-injection, exfiltration, malware and licensing risks. The configuration can restrict domains and HTTP methods, but unrestricted access is available, so the mere phrase “runs in a container” is not enough to determine safety. ([internet access](https://learn.chatgpt.com/docs/cloud/internet-access))

### Google Antigravity

Antigravity 2.0 works with local project folders and local agents. Its settings support project/worktree choices, terminal execution policies, outside-folder Allow/Ask/Deny choices and a sandbox mode. The documented default permission pattern allows workspace read/write while asking for non-workspace access, external tools, web activity and other broader operations. ([settings](https://antigravity.google/docs/settings), [permissions](https://antigravity.google/docs/permissions))

For the Antigravity CLI, Google describes its native sandbox as operating-system containment rather than a separate virtual machine or heavy container. It uses nsjail on Linux, sandbox-exec on macOS and AppContainer on Windows, and is disabled by default in the CLI. Google also documents a one-command unsandboxed escape, so a user can deliberately step outside the boundary. ([CLI sandbox](https://antigravity.google/docs/cli/sandbox))

Antigravity’s general permissions page describes terminal sandbox support on macOS and Linux as preview and says Windows is “coming soon,” while the CLI sandbox page documents Windows AppContainer support. This appears to describe different surfaces or documentation versions; the website should not flatten it into “Antigravity is sandboxed on all platforms.” ([permissions](https://antigravity.google/docs/permissions), [CLI sandbox](https://antigravity.google/docs/cli/sandbox))

Model Context Protocol connections can give Antigravity access to local or remote APIs. Their calls are permission-gated, and OAuth tokens are stored locally, but the connected service processes whatever information is sent under its own rules. ([MCP integrations](https://antigravity.google/docs/mcp), [Antigravity terms](https://antigravity.google/terms))

## Data, retention, training and human access

### Claude consumer accounts: Free, Pro and Max

Anthropic defines consumer inputs broadly enough to include chat, coding and agentic sessions, uploaded, local and connected content, and actions conducted through connected services. ([Anthropic Privacy Policy](https://www.anthropic.com/legal/privacy))

Consumer chats and coding sessions can be used for model improvement when the user enables Model Improvement. Safety-flagged conversations, explicit feedback and separately opted-in testing programs are exceptions; Incognito chats are not used for model improvement. ([consumer training policy](https://privacy.claude.com/en/articles/10023580-is-my-data-used-for-model-training), [consumer retention](https://privacy.claude.com/en/articles/10023548-how-long-do-you-store-my-data))

Deleted conversations disappear from visible history immediately and are deleted from backend systems within 30 days. De-identified material placed into training pipelines when Model Improvement is enabled may be kept for up to five years; safety-flagged inputs/outputs may be kept for two years, classifier scores for seven years and explicit feedback for five years. ([consumer retention](https://privacy.claude.com/en/articles/10023548-how-long-do-you-store-my-data))

Anthropic says employees cannot access consumer conversations by default, with exceptions for content deliberately shared as feedback and need-to-know Usage Policy review by designated Trust and Safety staff. Consumer information can be transferred to the United States or other countries outside Australia. ([conversation access](https://privacy.claude.com/en/articles/10458704-how-does-anthropic-protect-the-personal-data-of-claude-users), [Privacy Policy](https://www.anthropic.com/legal/privacy))

### Claude Team and Enterprise

Anthropic says commercial chats and coding sessions are not used for model training by default; exceptions include explicit feedback, opt-in or the Development Partner Program. ([commercial training policy](https://privacy.claude.com/en/articles/7996885-how-do-you-use-personal-data-in-model-training))

Saved commercial chats remain available so they can be resumed. Deleted chats disappear from history immediately and are removed from backend systems within 30 days. Enterprise owners can configure organisation retention from 30 days upward; Anthropic documents the default, if no custom policy is set, as indefinite. ([commercial retention](https://privacy.claude.com/en/articles/7996866-how-long-do-you-store-my-organization-s-data), [Enterprise retention controls](https://privacy.claude.com/en/articles/10440198-configure-custom-data-retention-controls-for-enterprise-plans))

Commercial data is stored in the United States by default. Processing may route through the United States, Europe, Asia and Australia, and safety, support or incident-response work may occur where Anthropic and its affiliates operate. ([server locations](https://privacy.claude.com/en/articles/7996890-where-are-your-servers-located-do-you-host-your-models-on-eu-servers))

Current Cowork activity is not included in audit logs, the Compliance API or data exports, although Team and Enterprise can use OpenTelemetry monitoring. Cowork is not covered by Anthropic’s Business Associate Agreement. ([Cowork architecture](https://support.claude.com/en/articles/14479288-claude-cowork-architecture-overview), [BAA coverage](https://privacy.claude.com/en/articles/8114513-business-associate-agreements-baa-for-commercial-customers))

### ChatGPT consumer accounts: Free, Go, Plus and Pro

For personal ChatGPT services, OpenAI says content may be used to train models. Free, Plus and Pro users can turn off “Improve the model for everyone” in Data Controls; OpenAI documents personal-plan sharing as enabled by default. ([data-use policy](https://help.openai.com/en/articles/5722486-how-your-data-is-used-to-improve-model-performance), [training controls](https://help.openai.com/en/articles/8983130))

Temporary Chats do not appear in history, do not create memories and are not used for training, but OpenAI can retain them for up to 30 days and may review them for abuse. Explicit feedback can include the whole associated conversation even when general training is disabled. Codex also has a separate control for sharing full environments that is not governed by the general ChatGPT toggle. ([Data Controls FAQ](https://help.openai.com/en/articles/7730893-datacontrols-faq), [data-use policy](https://help.openai.com/en/articles/5722486-how-your-data-is-used-to-improve-model-performance))

Ordinary saved chats remain until the user deletes them. Deleted chats are scheduled for permanent deletion within 30 days unless the content has already been de-identified or must be retained for security or legal reasons. Files saved to ChatGPT’s Library can persist independently where that feature applies, so deleting a local copy is not necessarily deletion of the stored ChatGPT copy. ([chat deletion](https://help.openai.com/en/articles/8809935-how-to-delete-and-archive-chats-in-chatgpt), [file storage and Library](https://help.openai.com/en/articles/20001052-file-storage-and-library-in-chatgpt))

OpenAI says limited authorised personnel and service providers can access content when necessary for abuse/security, support, legal matters or model improvement where the user has not opted out; access is controlled and monitored. ([agent privacy](https://help.openai.com/en/articles/11752874))

### ChatGPT Business, Enterprise and Edu

OpenAI says Business, Enterprise, Edu and API content is not used to train models by default. ([enterprise privacy](https://openai.com/enterprise-privacy/), [training controls](https://help.openai.com/en/articles/8983130))

OpenAI documents encryption at rest with AES-256 and in transit with TLS 1.2 or later. Business administrators can access, export and delete workspace conversations; Enterprise/Edu administrators can configure retention, and deleted or unsaved content is generally removed within 30 days subject to legal/security exceptions. ([enterprise privacy](https://openai.com/enterprise-privacy/))

OpenAI personnel and approved service providers can still have limited access for support, abuse/security investigation and legal obligations. Business/Enterprise controls therefore reduce training exposure but do not mean “no human access under any circumstances.” ([enterprise privacy](https://openai.com/enterprise-privacy/))

### Google Antigravity individual accounts

Google’s Antigravity terms say the service records and stores user data, interaction data, metadata and feedback while the service is running. It may use interactions to evaluate, develop and improve Google/Alphabet products, services and machine-learning technologies, and employees or contractors may access and review them; users can opt out in settings. ([Antigravity terms](https://antigravity.google/terms), [settings](https://antigravity.google/docs/settings), [FAQ](https://antigravity.google/docs/faq))

Google’s terms also place responsibility on the user for supervising agent actions and access, and state that third-party or open-source models can be subject to their own terms. ([Antigravity terms](https://antigravity.google/terms))

**Unverified — do not publish as fact:** a fixed retention period for individual Antigravity interactions; a guarantee that individual Antigravity model processing stays in Australia; a claim that disabling telemetry means Google retains no operational, security or billing data. The checked first-party Antigravity pages do not establish those propositions.

### Gemini consumer chat is a different data regime

Gemini Apps activity settings should not be presented as though they automatically govern Antigravity. For Gemini Apps, Google says Keep Activity is on by default for adults; chats and uploads may be used to improve services and models, with a subset reviewed by humans. Reviewed chats are disconnected from the account and can be retained for up to three years. ([Gemini Privacy Hub](https://support.google.com/gemini/answer/13594961?hl=en), [activity controls](https://support.google.com/gemini/answer/13278892))

With Gemini Apps Activity off, future chats are not used for model training unless the user submits feedback, but they are still retained for up to 72 hours for service and safety purposes and may be human-reviewed for protection. Turning activity off also disables some connected-app features. ([Gemini Privacy Hub](https://support.google.com/gemini/answer/13594961?hl=en), [activity controls](https://support.google.com/gemini/answer/13278892))

### Antigravity for organisations through Google Cloud

Google says organisation deployments use Gemini Enterprise Agent Platform in the organisation’s own Google Cloud project, under Google Cloud terms, with consumption billing, private-networking and data-residency options. The individual Antigravity IDE is not part of that enterprise offer. ([Antigravity enterprise](https://antigravity.google/docs/enterprise))

For managed generative-AI models on Google Cloud, Google says it does not use customer data to train or fine-tune models without prior permission or instruction. ([Gemini Enterprise locations](https://docs.cloud.google.com/gemini/enterprise/docs/locations?hl=en), [Google Cloud data-residency list](https://cloud.google.com/terms/data-residency))

**Unverified — do not publish as fact:** that every Antigravity enterprise task qualifies for zero data retention; that all Antigravity Agent Platform features support Australian in-country storage and processing; that enabling one residency control covers third-party connectors. The official Google Cloud pages describe feature-specific exceptions.

## Practical confidentiality guidance for translators and interpreters

The following is an **editorial risk-management recommendation**, not a legal opinion or a vendor certification.

1. **Use synthetic, public or expressly authorised material in the lecture.** Consumer accounts can involve model-improvement settings, human safety review and overseas processing, and agents can pass content to connected services. ([Anthropic consumer policy](https://www.anthropic.com/legal/privacy), [OpenAI data use](https://help.openai.com/en/articles/5722486-how-your-data-is-used-to-improve-model-performance), [Antigravity terms](https://antigravity.google/terms))
2. **Ask whose account and contract govern the job.** A personal subscription is not equivalent to a Team, Business or Enterprise agreement, and the three vendors document different default training and retention rules for consumer and commercial offerings. ([Claude commercial training](https://privacy.claude.com/en/articles/7996885-how-do-you-use-personal-data-in-model-training), [OpenAI enterprise privacy](https://openai.com/enterprise-privacy/), [Antigravity enterprise](https://antigravity.google/docs/enterprise))
3. **Check the actual settings before every demonstration.** Relevant settings include training/model-improvement controls, Incognito or Temporary Chat, connected folders, permission mode, computer use, internet access, connectors, credit overages and auto-reload. ([Claude usage credits](https://support.claude.com/en/articles/12429409-manage-usage-credits-for-paid-claude-plans), [ChatGPT Data Controls](https://help.openai.com/en/articles/7730893-datacontrols-faq), [ChatGPT permission modes](https://learn.chatgpt.com/docs/permission-modes), [Antigravity settings](https://antigravity.google/docs/settings))
4. **Treat visible desktop content as exposed when computer use is enabled.** Anthropic says Claude takes screenshots and has no sandbox between itself and approved apps; other agents with browser/app access similarly have broader action and data paths than a command-only sandbox. ([Claude computer use](https://support.claude.com/en/articles/14128542-let-claude-use-your-computer-in-cowork), [OpenAI agent security](https://learn.chatgpt.com/docs/agent-approvals-security), [Antigravity permissions](https://antigravity.google/docs/permissions))
5. **Check the connected service as well as the AI vendor.** Connectors can send information to third parties that apply their own privacy and retention rules. ([Claude connectors](https://support.claude.com/en/articles/11176164-use-connectors-to-extend-claude-s-capabilities), [OpenAI agent security](https://learn.chatgpt.com/docs/agent-approvals-security), [Antigravity MCP](https://antigravity.google/docs/mcp))
6. **Do not interpret a sandbox as a confidentiality guarantee.** A sandbox limits what executed code can reach; it does not prevent authorised files from being sent for model processing or stop a permitted connector/app from acting on data. ([Claude Cowork safety](https://support.claude.com/en/articles/13364135-use-claude-cowork-safely), [OpenAI sandboxing](https://learn.chatgpt.com/docs/sandboxing), [Antigravity CLI sandbox](https://antigravity.google/docs/cli/sandbox))
7. **Use the least-permissive mode that still completes the task.** OpenAI recommends approval-gated access over Full access; Google documents Deny/Ask/Allow controls; Anthropic provides manual permission modes. ([OpenAI permission modes](https://learn.chatgpt.com/docs/permission-modes), [Antigravity permissions](https://antigravity.google/docs/permissions), [Claude Cowork guide](https://support.claude.com/en/articles/13345190-get-started-with-claude-cowork))
8. **Delete deliberately, but do not promise immediate backend erasure.** Each vendor documents retention windows and exceptions, and files or reviewed/training material can follow a different lifecycle from the visible chat. ([Claude retention](https://privacy.claude.com/en/articles/10023548-how-long-do-you-store-my-data), [ChatGPT deletion](https://help.openai.com/en/articles/8809935-how-to-delete-and-archive-chats-in-chatgpt), [Gemini Privacy Hub](https://support.google.com/gemini/answer/13594961?hl=en))

## Appendix: model choice, subscription usage and token prices

### Recommendation for the audience-facing cost section

Keep the main lecture page focused on **the consumer subscription price, what the subscription includes, when it stops, and whether optional overage is enabled**. The audience is not buying API access, so API token prices should sit in an optional drop-down or presenter note rather than in the main plan comparison.

A useful one-sentence lesson is:

> Some models consume paid credits faster than others. Use the most capable model when the task needs it, but a faster or more economical model may be enough for routine work. The exact effect on an included subscription allowance is not always published.

This is more accurate than “always choose the cheapest model.” A higher-priced model can sometimes finish with fewer turns or less correction, so per-token price alone cannot predict the total cost of a completed task. Anthropic describes Fable as intended for ambitious long-running work and Sonnet as its high-performance coding/agent model, while its usage guide says conversation length, attachments, tools, model choice, effort and artifact creation all affect plan usage. ([Fable 5](https://www.anthropic.com/claude/fable), [Claude pricing](https://claude.com/pricing), [usage-limit factors](https://support.claude.com/en/articles/9797557-usage-limit-best-practices))

### Fable is a real model

**Claude Fable 5 is an official current Anthropic model**, released in June 2026 and available in Claude applications and the Claude API. It is not a mistaken reference to Opus or Haiku. Anthropic positions Fable for its most ambitious, long-running knowledge-work and coding tasks and publishes the API identifier `claude-fable-5`. ([Fable product page](https://www.anthropic.com/claude/fable), [models overview](https://platform.claude.com/docs/en/about-claude/models/overview), [Claude Code model selection](https://support.claude.com/en/articles/11940350-claude-code-model-configuration))

Availability does not mean inclusion. Under Anthropic’s current rule, effective 20 July 2026, Fable is included for Max subscribers for up to 50% of their shared weekly usage limit and it uses that limit faster than other Claude models. On Pro, Fable is not part of the included allowance and uses pay-as-you-go usage credits from the first request. The same rule applies across Claude web, mobile, Desktop, Cowork and Claude Code. ([Fable on your plan](https://support.claude.com/en/articles/15424964-claude-fable-5-on-your-plan))

Fable also carries a non-price caveat that matters for this audience: Anthropic designates it a Covered Model and requires prompts and completions to be retained for at least 30 days by default, including on commercial and third-party surfaces; zero-data-retention treatment is not available for Fable. ([Covered Models](https://support.claude.com/en/articles/15425695-covered-models))

### The “Sonnet over Fable” example

The example is valid if it is phrased carefully. At the checked rates, Fable 5 costs five times Sonnet 5’s introductory input and output price. After Sonnet’s introductory period ends, Fable’s published rate will be about 3.33 times Sonnet’s input and output rate. That comparison applies directly to API-billed use, to Fable on Pro because it uses credits from the start, and to deliberately enabled Claude usage credits after other included limits are exhausted. ([Claude pricing](https://claude.com/pricing), [Claude usage credits](https://support.claude.com/en/articles/12429409-manage-usage-credits-for-paid-claude-plans), [Fable plan rules](https://support.claude.com/en/articles/15424964-claude-fable-5-on-your-plan))

**Recommended lecture wording:**

> Fable is Claude’s strongest model for very demanding, long-running work, but Sonnet is much less expensive when paid credits are being used and is designed for strong everyday agent and coding work. Start with the model that is sufficient for the task, and move up when the task genuinely needs more capability. We cannot turn the API price difference into an exact promise about how long the included subscription allowance will last.

Do not present the API table as though attendees will receive a token bill while staying inside an ordinary consumer subscription. Show it only to explain why the model selector can matter once usage is constrained or optional paid credits are active.

## Suggested website implementation notes

These are editorial recommendations derived from the evidence above.

- Put a **Check current price** button beside every price, not only in a footer.
- Add a visible “Checked 22 July 2026” stamp to pricing, feature and privacy panels.
- Use expandable sections for **Where does the task run?**, **What can it access?**, **Can it spend more?**, **Where can the data go?**, **Can it be used for training?**, **How long can it remain?**, and **Can a person review it?**
- Keep consumer and business/enterprise data rules in separate tabs; combining them produces materially misleading statements.
- Put the wording “A sandbox limits executed code; it is not a confidentiality guarantee” beside, not underneath, any virtual-machine or container diagram.
- Give each product a direct download link, a direct price link and a direct privacy link.
- Keep the documentation-conflict warnings available to the presenter even if they are hidden from the ordinary scrolling page.
